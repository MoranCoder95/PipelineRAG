Why PipelineRAG?

Modular Design

Build RAG applications by connecting independent, reusable nodes. Each component has a single responsibility and clear interface.

Simplified Development

Significantly reduces development complexity compared to traditional frameworks. Focus on business logic, not framework intricacies.

Pipeline-Native

Built from the ground up using the Pipeline design pattern. Natural fit for RAG workflows with clear data flow.

Flexible Architecture

Supports complex DAG structures, conditional routing, and custom components. Easy to extend and customize.

Unified Data Flow

PipelineData provides standardized communication between all components. Consistent, type-safe data handling.

Easy Learning Curve

Intuitive API design with clear abstractions. Developers report significantly better learning experience compared to traditional frameworks.

Quick Example

Building a complete RAG pipeline with actual GitHub code

üìÑ Document Indexing Pipeline

from pipeline_data import PipelineData
from pipelines.base import Pipeline
from nodes.document.pdf_processor import PDFProcessorNode
from nodes.document.text_splitter import TextSplitterNode
from nodes.embedding.text_embedding import TextEmbeddingNode
from nodes.vector.vector_store_node import VectorStoreNode

# Create indexing pipeline
pipeline = Pipeline()
pipeline.add_node(PDFProcessorNode(), "pdf_processor", ["File"])
pipeline.add_node(TextSplitterNode(chunk_size=256, chunk_overlap=50), 
                  "text_splitter", ["pdf_processor"])
pipeline.add_node(TextEmbeddingNode(), "text_embedding", ["text_splitter"])
pipeline.add_node(VectorStoreNode("data/index.bin", "data/mapping.pkl"), 
                  "vector_store", ["text_embedding"])

# Process documents
pdf_files = ["doc1.pdf", "doc2.pdf"]
result = pipeline.run(file_paths=pdf_files)
